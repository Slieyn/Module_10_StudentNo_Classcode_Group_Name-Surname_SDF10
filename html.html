Question 1:Application Programming Interface

An API is a middleman between software applications, allowing them to communicate with one another. It's significance in software development is that it simplifys the creation of e.g, Web,desktop, mobile and how they work. Data is now shared easily, with guaranteed security. APIs are able to make certain information private; as well as enhancing innovation and scalability as users are able to use these web applications and apps on desktops and mobiles.

Example: Resturant

Waiter >> API >> Kitchen >> Final Product

(Point of contact >> POS Machine >> Order Placed >> Dish)

In this example a customer walks into a resturant, he sits done and is given a menu to choose from. A waiter comes back and takes his order, writes it on a notebook. The waiter orders via the POS(Point of Sale Machine),which then produces a slip to the kitchen staff(Chef) listing what the cutomer has ordered. Now, bare in mind that the customer does not have any idea of how his food is made,he only cares about the final product. The Kitchen staff plays a huge role here because they are the ones that need to have the correct ingrediants and methods on how to create the dish. Once the dish has been prepared, the waiter then takes the dish to the table, and the customer is served

Question 3:Components and Types of APIs

Main Components

  • EndPoint An Endpoint is Locator. For example,when a clients requests a resource, they do it via an endpoint. Which effectively states that the resource is available in a specific location.

    • Method

  • These define the types of actions that can be executed against APIs Endpoints. Here are examples of the HTTP Method (HyperText Transfer Protocol)
  • GET:Used to get data.
  • POST: Used to send data to the server.
  • PUT: Updates existing data.
  • PATCH: Used when changing certain information for an email and not the actual user name.
  • DELETE: Removing a resource

    • DataFormats

  • DataFormats are communicators.They usualy communicate using JSON. JSON(JavaScript Object Notation) is a lightweight data format that easy to read and write. It's role is simplicity and effienency.

Types of APIs

  • WEB APIs, I call them Remote APIs
    • These types of APIs operate and facilitate over the internet. For example, Shazam,that is an app that allows us to identify a song by sound waves providing information, e.g the name of the song and the artist.
  • REST OR RESTful, This type of API acts as a hat. "Representational State Transfer" REST allows the server to cater to mulltiple types of clients e.g mobile,web and desktop. The main theme is "The serparations of concerns" Meaning the client(browser) and the server(stores and retrives date) are kept separate from each other. Reason being for the browser to focus on the user interface while the server handles the data. "This principle is called the Client Server-Architecture" Every request from a client contains all the information the server needs to understand and process the request. No session data is stored on the server between requests. "This principle is called Statelessness" I find REST/RESTful useful because I find it useful! It enables scalability meaning there won't be any storing of information.Enhances simplicty, allowing requests to be handled independently and flexability, allowing clients to send requests in any order without relying on prior requests.
